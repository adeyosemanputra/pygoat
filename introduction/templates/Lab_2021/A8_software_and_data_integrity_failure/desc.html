What is Software and Data Integrity Failures

Software and data integrity failures relate to code and infrastructure that does not protect against integrity violations. Insecure deserialization, untrusted updates, and tampered plugins are common examples.

Role stored in a base64 + pickle cookie. Flip the admin bit to access restricted content.

Malicious executable download through tampered update pipeline.

Insecure Deserialization

Applications are vulnerable when they deserialize hostile or tampered objects supplied by an attacker.

How to Prevent

  • Verify integrity of serialized objects
  • Restrict allowed classes during deserialization
  • Run deserialization in low privilege environments
  • Log and monitor deserialization failures
  • Validate update sources and signatures
