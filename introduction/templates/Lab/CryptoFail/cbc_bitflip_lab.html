Bit-Flipping Helper Tool

Use this Python script to perform the bit-flip attack:


import base64

def flip_bit(cookie_b64, position, flip_value):
    # Decode cookie
    cookie = bytearray(base64.b64decode(cookie_b64))
    
    # Flip the bit at position
    cookie[position] ^= flip_value
    
    # Re-encode
    return base64.b64encode(cookie).decode()

# Your cookie here
original = "YOUR_COOKIE_HERE"

# To flip 'admin=0' to 'admin=1'
# Position depends on username length and AES block boundaries (CBC uses the previous block).
# For a 4-char username, '0' falls at the start of the second block, so flip byte 15 (0-based) in the ciphertext.
# XOR with: ord('0') ^ ord('1') = 1

modified = flip_bit(original, 15, 1)

Hints & Walkthrough
Hint 1: Understanding the Cookie Structure

The plaintext format is: user=USERNAME;admin=0
After padding and encryption with AES-CBC (16-byte blocks):
- Block 1: user=USERNAME;adm
- Block 2: in=0 (plus padding)

Hint 2: CBC Decryption Property

In CBC decryption: Plaintext[i] = Decrypt(Ciphertext[i]) XOR Ciphertext[i-1]
This means modifying Ciphertext[0] will affect Plaintext[1]!

Hint 3: Finding the Right Position

For a 4-character username like "test":
- Plaintext block 1: user=test;admin= (16 bytes)
- Plaintext block 2: 0 + padding
The '0' in 'admin=0' is at position 0 of block 2.
To affect block 2, modify the corresponding byte in block 1 (ciphertext).

Hint 4: The XOR Trick

To change '0' (0x30) to '1' (0x31):
0x30 XOR 0x01 = 0x31
So flip bit with value 1 at the position corresponding to 'admin=0'

Hint 5: Step-by-Step Attack
  1. Create account with 4-character username (e.g., "test")
  2. Get your session cookie (base64 encoded)
  3. Decode from base64 to bytes
  4. Find position of the byte that affects 'admin=0'
  5. XOR that byte with 1
  6. Re-encode to base64
  7. Submit modified cookie
Hint 6: Python Solution 

import base64

original_cookie = "YOUR_COOKIE_HERE"
cookie_bytes = bytearray(base64.b64decode(original_cookie))

# For 4-char username, the critical byte is around position 15
# This affects the first byte of block 2 during decryption
cookie_bytes[15] ^= 1  # Flip 0x30 to 0x31

modified_cookie = base64.b64encode(cookie_bytes).decode()
print(modified_cookie)
Why This Attack Works:
  • No Integrity Protection: CBC mode provides confidentiality but NOT authenticity
  • XOR Properties: The XOR in CBC decryption allows predictable modifications
  • Known Plaintext: Attacker knows the cookie structure
  • Block Cipher Math: Changing ciphertext block N affects plaintext block N+1
Proper Defense:
# Use AES-GCM (authenticated encryption)
from Crypto.Cipher import AES

cipher = AES.new(key, AES.MODE_GCM)
ciphertext, tag = cipher.encrypt_and_digest(data)

# Verification will fail if tampered!
plaintext = cipher.decrypt_and_verify(ciphertext, tag)
