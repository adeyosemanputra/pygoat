🔐 Insecure Cryptographic Storage Lab - Hardcoded Keys

📋 Scenario:

You've gained access to a company's database backup and their source code repository. The database contains encrypted sensitive information. Your goal is to decrypt it and find the hidden flag.

# Encryption utilities for sensitive data
from cryptography.fernet import Fernet
import base64

# TODO: Move this to environment variable
ENCRYPTION_KEY = b'{{ encryption_key }}'

def encrypt_data(plaintext):
    """Encrypt sensitive data"""
    cipher = Fernet(ENCRYPTION_KEY)
    encrypted = cipher.encrypt(plaintext.encode())
    return encrypted.decode()

def decrypt_data(ciphertext):
    """Decrypt sensitive data"""
    cipher = Fernet(ENCRYPTION_KEY)
    decrypted = cipher.decrypt(ciphertext.encode())
    return decrypted.decode()

⚠️ Impact of Hardcoded Encryption Keys:
  • Complete Data Breach: All encrypted data can be decrypted instantly
  • Cannot Rotate Keys: Changing the key requires code changes and redeployment
  • Version Control Exposure: Keys remain in Git history even if "deleted"
  • Insider Threats: Any developer with code access can decrypt production data
  • Third-Party Access: Contractors, offshore teams get the key
✅ Proper Solution:
  • Store keys in AWS Secrets Manager, Azure Key Vault, or HashiCorp Vault
  • Use environment variables (minimum - not in committed .env files!)
  • Implement key rotation policies
  • Apply principle of least privilege for key access
  • Use git-secrets to prevent accidental commits
