Scenario 1: Weak Encryption Algorithm

The Vulnerability

This application uses MD5 hashing for password storage. MD5 is cryptographically broken and unsuitable for password hashing because:

  • It's extremely fast - billions of hashes can be computed per second
  • Rainbow tables exist for common passwords
  • No built-in salt mechanism
  • Collision vulnerabilities discovered in 1996

Attack Steps:
  1. Attacker gains database access (SQL injection, insider threat, or breach)
  2. Extracts password hashes from the user table
  3. Cracks hashes using:
    • Rainbow tables (precomputed hash databases)
    • Online hash lookup services (e.g., crackstation.net)
    • GPU-accelerated brute force (hashcat, John the Ripper)
  4. Gains plaintext passwords in seconds to minutes
Time-to-Crack Comparison:
Algorithm Password: "admin123" Password: "P@ssw0rd!" Security Level
MD5 < 1 second < 1 second ❌ Broken
SHA-1 < 1 second < 1 second ❌ Deprecated
bcrypt ~12 hours ~3 years ✅ Secure
Argon2 ~24 hours ~5 years ✅ Most Secure 

# ❌ VULNERABLE CODE - DO NOT USE IN PRODUCTION
import hashlib

def register_user(username, password):
    # Weakness: Using MD5 for password hashing
    password_hash = hashlib.md5(password.encode()).hexdigest()
    
    # Save to database
    user = User(username=username, password=password_hash)
    user.save()
    
def login_user(username, password):
    password_hash = hashlib.md5(password.encode()).hexdigest()
    user = User.objects.filter(username=username, password=password_hash).first()
    return user
      

# ✅ SECURE CODE - Recommended Approach
from django.contrib.auth.hashers import make_password, check_password

def register_user(username, password):
    # Uses PBKDF2 with SHA-256 by default (Django 4.x)
    # Can be configured to use Argon2 (recommended)
    password_hash = make_password(password)
    
    user = User(username=username, password=password_hash)
    user.save()
    
def login_user(username, password):
    user = User.objects.filter(username=username).first()
    if user and check_password(password, user.password):
        return user
    return None

# Alternative: Using Argon2 (install argon2-cffi)
from argon2 import PasswordHasher

ph = PasswordHasher()

def register_user_argon2(username, password):
    password_hash = ph.hash(password)
    user = User(username=username, password=password_hash)
    user.save()

Mitigation

Recommended Solutions:

  • ✅ Use bcrypt or Argon2 for password hashing
  • ✅ Configure Django to use Argon2: PASSWORD_HASHERS = ['django.contrib.auth.hashers.Argon2PasswordHasher', ...]
  • ✅ Implement automatic hash upgrades when users log in
  • ✅ Use strong salt and appropriate work factor/iterations
  • ✅ Never store passwords in plaintext or reversible encryption
