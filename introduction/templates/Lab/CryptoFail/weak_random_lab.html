    # VULNERABLE: Using time as seed!
    seed = int(time.time())
    random.seed(seed)
    
    chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789'
    token = ''.join(random.choice(chars) for _ in range(16))
    
    return token

Why This Attack Works:
  1. Predictable Seed: Using timestamp as seed makes it easily guessable
  2. Weak PRNG: Python's random module is not cryptographically secure
  3. Deterministic Output: Same seed always produces same sequence
  4. Limited Seed Space: Timestamps have limited precision (1 second)
Secure Alternative:
import secrets

# Cryptographically secure random token
token = secrets.token_urlsafe(32)  # 256 bits of entropy

# OR using secrets.choice
import string
chars = string.ascii_letters + string.digits
token = ''.join(secrets.choice(chars) for _ in range(32))
