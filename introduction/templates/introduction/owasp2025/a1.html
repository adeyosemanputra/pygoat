A1: Broken Access Control

What is Broken Access Control?

Broken access control means users can perform actions or view data outside their intended permissions.

Real World Example

An attacker changes URL from /user/123/profile to /user/124/profile and accesses another user's data.

Impact

  • Unauthorized access to sensitive data
  • Privilege escalation
  • Data modification or deletion

Mitigation

  • Deny access by default
  • Implement server-side access control checks
  • Log access control failures
