A8: Vulnerable and Outdated Components

What is this vulnerability?

This category covers reliance on frameworks, libraries, and services with known vulnerabilities or end-of-life support. Attackers exploit public CVEs in dependencies you did not write, including deserialization bugs, RCE in parsers, or default credentials on bundled services.

Real World Example

The Equifax breach involved exploitation of a disclosed vulnerability in Apache Struts. A public patch existed, but the vulnerable component remained exposed—showing that unpatched dependencies can compromise entire organizations.

Impact

  • Remote code execution or data leakage through library flaws
  • Cascade effects when one weak component exposes the rest of the stack
  • Compliance findings when unsupported versions remain in production

Mitigation

  • Maintain a software bill of materials (SBOM) and patch on a defined cadence
  • Run software composition analysis in CI/CD and track vendor security advisories
  • Remove unused dependencies and isolate third-party services with network controls
