A5: Injection

What is this vulnerability?

Injection flaws let attackers supply malicious input that is interpreted as part of a command or query—such as SQL, OS commands, LDAP, or server-side template code. The application fails to separate untrusted data from the interpreter, leading to execution of attacker-controlled logic.

Real World Example

The Heartland Payment Systems breach involved attackers using SQL injection to gain a foothold and deploy malware that captured card data. SQL injection has been a leading cause of large-scale data theft in web applications for years.

Impact

  • Full database access, data exfiltration, or data destruction
  • Remote code execution when command or template injection is possible
  • Authentication bypass and escalation of privilege

Mitigation

  • Use parameterized queries and prepared statements for database access
  • Avoid passing user input to shells; use safe APIs with strict allow-lists
  • Apply context-appropriate encoding and validate input against expected formats
