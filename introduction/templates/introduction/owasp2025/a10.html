A10: Mishandling of Exceptions

What is this vulnerability?

Mishandling exceptions means errors are exposed to users in ways that leak stack traces, file paths, SQL fragments, or framework details; swallowed without logging; or handled so broadly that security checks are skipped in failure paths. Inconsistent error handling can also leave resources or sessions in an unsafe state.

Real World Example

Verbose framework debug pages in production—such as detailed ASP.NET or Django error screens—have historically revealed versions, paths, and queries that attackers used to refine exploits. Error messages that echo user input can also introduce cross-site scripting.

Impact

  • Information disclosure that aids targeted exploitation and fingerprinting
  • Logic or authorization gaps when exceptions bypass normal control flow
  • Compliance issues when error output contains personal or regulated data

Mitigation

  • Show generic messages to users; log detailed errors only on the server
  • Disable debug modes in production and configure secure global error handlers
  • Ensure cleanup and security checks run in finally blocks or equivalent patterns
