A4: Cryptographic Failures

What is this vulnerability?

Cryptographic failures arise when sensitive data is not protected with appropriate encryption, hashing, or key management. Examples include weak or obsolete algorithms, hardcoded keys, missing TLS, improper password storage, or encrypting data but exposing the keys in the same repository or environment.

Real World Example

Many breaches involve stolen databases where passwords were hashed with weak schemes or not salted, allowing attackers to recover credentials offline. Poor TLS configuration has also enabled interception of session tokens and credentials on untrusted networks.

Impact

  • Exposure of passwords, tokens, personal data, and payment information
  • Account takeover and fraud using recovered secrets
  • Regulatory fines and loss of trust when confidentiality is breached

Mitigation

  • Use vetted libraries and modern algorithms; avoid rolling your own cryptography
  • Store keys in dedicated secret managers or HSMs; rotate and scope keys by purpose
  • Enforce TLS for data in transit; use slow salted password hashing (e.g., Argon2) for passwords at rest
