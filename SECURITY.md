# Security Policy

## Supported Versions

PyGoat is an intentionally vulnerable application designed for security training.
The following versions are currently maintained:

| Version | Supported |
| ------- | --------- |
| latest (main) | ✅ |
| older releases | ❌ |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

PyGoat is a deliberately vulnerable application, so most vulnerabilities are
intentional and part of the learning experience. However, if you discover an
unintended vulnerability in the platform infrastructure itself (e.g. the Django
app framework, Docker setup, or CI pipeline), please report it responsibly.

### How to Report

Email the OWASP PyGoat maintainers at:
📧 **pygoat@owasp.org**

### What to Include

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Your suggested fix (optional but appreciated)

## Response Timeline

- **Acknowledgement:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Resolution or mitigation plan:** Within 30 days

## Responsible Disclosure

We follow a coordinated disclosure policy. Please give us reasonable time to
address the issue before any public disclosure. We appreciate researchers who
act in good faith and will acknowledge contributions in release notes.
