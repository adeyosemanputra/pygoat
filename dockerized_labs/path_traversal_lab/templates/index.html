What is Path Traversal?

Path traversal (also known as directory traversal) is a vulnerability that allows an attacker to read arbitrary files on the server by manipulating file paths. When a web application uses user-supplied input to construct file paths without proper validation, an attacker can inject special characters like ../ (dot-dot-slash) to escape the intended directory and access files elsewhere on the filesystem.

This is classified under A01:2021 – Broken Access Control in the OWASP Top 10 because the application fails to enforce proper access boundaries on the files a user can retrieve.

Lab Scenario

You are a user of a company's internal document portal. The portal lets you browse and download shared files from the /uploads directory on the server.

However, the IT team stored some admin credentials in a /secret directory that is supposed to be off-limits. Your goal is to exploit the download feature to read those credentials.

Hint: Look at the URL when you download a file. The file parameter is passed directly to the server. What happens if you replace the filename with a path that includes ../ sequences?

Real-World Impact

  • Reading sensitive configuration files (/etc/passwd, .env)
  • Accessing application source code and discovering further vulnerabilities
  • Leaking credentials, API keys, and database connection strings
  • In severe cases, writing to arbitrary paths to achieve remote code execution

Mitigation

  • Never use raw user input to build file paths
  • Resolve the canonical/real path and verify it stays within the allowed directory
  • Use an allowlist of permitted filenames or map IDs to files on the server side
  • Strip or reject input containing .., /, or \
  • Run the application with least-privilege filesystem permissions
