🛡️ Security Fix for Path Traversal

Secure Implementation


import os
from flask import Flask, request, abort, send_file

UPLOAD_DIR = os.path.realpath("uploads")

@app.route('/download')
def download_secure():
    filename = request.args.get('file')
    if not filename:
        abort(400)

    # 1. Resolve the absolute, canonical path
    requested = os.path.realpath(
        os.path.join(UPLOAD_DIR, filename)
    )

    # 2. Confirm it is still inside the allowed directory
    if not requested.startswith(UPLOAD_DIR + os.sep):
        abort(403, "Path traversal blocked")

    # 3. Confirm the file exists
    if not os.path.isfile(requested):
        abort(404)

    return send_file(requested)

Why This Works

  • os.path.realpath() resolves symlinks and .. components, producing the true absolute path.
  • The startswith() check guarantees the resolved path is still inside the uploads folder.
  • Even encoded traversal variants (%2e%2e%2f, ..%5c) are caught because the OS resolves them before the check.

Solution Walkthrough

When you open the lab, you see a list of shared documents with download links.

Step 1 — Observe the URL pattern

Click any file and note the URL:

/download?file=report_q1.txt
Step 2 — Attempt path traversal

The file parameter is passed directly to the server without validation. Replace the filename with a relative path that escapes the uploads/ directory:

/download?file=../secret/admin_credentials.txt
Step 3 — Read the secret

The server resolves uploads/../secret/admin_credentials.txt to secret/admin_credentials.txt and returns its contents, including the admin credentials and the flag.

Try the Secure Endpoint

Now try the same payload against the secure endpoint:

/download/secure?file=../secret/admin_credentials.txt

You will get an "Access denied" message because the secure version validates the resolved path.
