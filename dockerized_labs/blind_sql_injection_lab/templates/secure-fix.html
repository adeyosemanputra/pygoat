Security Fix for Blind SQL Injection

Secure Implementation (Parameterized Query + Safe Responses)


def secure_login(username, password):
	conn = sqlite3.connect("database.db")
	cursor = conn.cursor()

	query = "SELECT username FROM users WHERE username = ? AND password = ?"
	user = cursor.execute(query, (username, password)).fetchone()
	conn.close()

	if user:
		return {"status": "ok"}

	# Never reveal query errors or condition-specific hints
	return {"status": "invalid_credentials"}

How to Prevent Blind SQL Injection

  • Use parameterized queries everywhere
  • Do not concatenate user input into SQL
  • Return uniform authentication messages
  • Avoid response differences that reveal boolean results
  • Add rate limits and account lockout controls
  • Log server-side errors securely for developers only
