🛡️ Security Fix for Exceptional Conditions

Secure Implementation Example


from decimal import Decimal, InvalidOperation
import logging

COUPON_RATES = {
	"SAVE10": Decimal("0.10"),
	"VIP25": Decimal("0.25"),
	"BETA05": Decimal("0.05"),
}

def safe_calculate(order_total_raw: str, divisor_raw: str, coupon_raw: str) -> str:
	try:
		order_total = Decimal(order_total_raw)
		divisor = Decimal(divisor_raw)
	except InvalidOperation:
		raise ValueError("Invalid numeric input")

	if divisor <= 0:
		raise ValueError("Divisor must be greater than zero")

	coupon = coupon_raw.strip().upper()
	if coupon not in COUPON_RATES:
		raise ValueError("Unknown coupon code")

	discount_rate = COUPON_RATES[coupon]
	discount_amount = order_total / divisor
	final_total = order_total - (discount_amount * discount_rate)
	return f"${final_total:.2f}"

@app.route('/lab-secure', methods=['POST'])
def lab_secure():
	try:
		result = safe_calculate(
			request.form.get("order_total", ""),
			request.form.get("divisor", ""),
			request.form.get("coupon", ""),
		)
		return render_template("lab.html", result=result)
	except ValueError as exc:
		return render_template("lab.html", error=str(exc))
	except Exception:
		logging.exception("Unexpected error in pricing calculator")
		return render_template("lab.html", error="Something went wrong. Please try again.")

Solution

Triggering edge cases in the lab returns stack traces and internal details. This is unsafe because it exposes file paths, framework versions, and business logic. The fix is to validate input and handle exceptions safely.

  • Validate numeric inputs and reject non-numeric values early.
  • Block invalid divisors (such as 0 or negative values).
  • Allowlist coupon codes instead of accessing dictionary keys directly.
  • Return generic error messages to the user.
  • Log full exception details server-side for debugging.
