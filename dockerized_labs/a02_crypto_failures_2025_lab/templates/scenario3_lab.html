Scenario 3: Weak Pseudorandom Number Generation – Lab

Objective: Predict the admin token by exploiting the weak random number generator seeded with a timestamp.

Leaked Source Code

import random
import time

# Token generation system
def generate_admin_token():
    # Seed with current time
    seed = int(time.time())
    random.seed(seed)
    token = random.randint(100000, 999999)
    return token

# Token generated at server startup
ADMIN_TOKEN = generate_admin_token()
TOKEN_TIMESTAMP = int(time.time())

Attack Strategy

Since the token is generated using time.time() as the seed, you can reproduce the exact same token:

  1. Use the known token generation timestamp from System Information
  2. Seed Python's random with that timestamp
  3. Call the same random function to get the identical token

Lesson Learned:

Using weak PRNGs with predictable seeds (like timestamps) makes security tokens completely insecure. Always use Python's secrets module for cryptographic operations.
