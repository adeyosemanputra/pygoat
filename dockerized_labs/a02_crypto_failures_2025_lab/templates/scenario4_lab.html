Attack Strategy

Cookie Structure:

Base64( [16 bytes IV] + [encrypted blocks] )

Plaintext: "user=alice;role=user"
                        ↑
                    Target: change "user" to "admin"

Bit-Flipping Math:

1. Know the plaintext at position P: 'u' (0x75)
2. Want to change it to: 'a' (0x61)
3. Calculate XOR difference: 0x75 XOR 0x61 = 0x14
4. Flip that bit in the IV at the same position
5. When decrypted: P XOR 0x14 = target character!

Python Attack Code:

import base64

# Your current cookie
cookie = "{{ cookie }}"
cookie_bytes = bytearray(base64.b64decode(cookie))

# Positions in "user=alice;role=user"
# We want to change "role=user" to "role=admin"
# Position of 'u' in "user" (after "role="): 17
# Position of 's' in "user": 18
# Position of 'e' in "user": 19  
# Position of 'r' in "user": 20

# XOR differences to change "user" -> "admin"
# u -> a: ord('u') ^ ord('a') = 0x75 ^ 0x61 = 0x14
# s -> d: ord('s') ^ ord('d') = 0x73 ^ 0x64 = 0x17
# e -> m: ord('e') ^ ord('m') = 0x65 ^ 0x6d = 0x08
# r -> i: ord('r') ^ ord('i') = 0x72 ^ 0x69 = 0x1b
# (last char stays 'n')

# Flip bits in IV (first 16 bytes) at position 17-20
cookie_bytes[17] ^= 0x14  # u -> a
cookie_bytes[18] ^= 0x17  # s -> d
cookie_bytes[19] ^= 0x08  # e -> m
cookie_bytes[20] ^= 0x1b  # r -> i

# Encode back to base64
modified_cookie = base64.b64encode(cookie_bytes).decode()
print(modified_cookie)

Hints

Hint 1: Understand the Structure

The cookie is Base64-encoded. The first 16 bytes after decoding are the IV. The IV is XORed with the first plaintext block during decryption.

Hint 2: Find the Target

You need to change "role=user" to "role=admin". Calculate the byte positions of 'u', 's', 'e', 'r' in the plaintext string "user=alice;role=user".

Hint 3: XOR Magic

For each character you want to change, calculate: original_char XOR target_char. Then XOR that value with the corresponding IV byte.

Lesson Learned:

CBC mode provides confidentiality but NOT integrity or authenticity. Always use authenticated encryption modes (AES-GCM) or add HMAC to detect tampering. Never trust decrypted data without verification!
