Welcome to the Cryptographic Failures Lab

About This Lab:

This lab demonstrates OWASP Top 10:2025 A02 - Cryptographic Failures through 4 interactive scenarios. Each scenario includes educational content, vulnerable code examples, and hands-on challenges.

Security Warning:

This application contains intentional security vulnerabilities for educational purposes only. DO NOT use any code patterns from this lab in production applications.

Lab Scenarios

Scenario 1: Weak Encryption Algorithm

Learn about the dangers of using deprecated hashing algorithms like MD5 for password storage.

Topics: MD5 weaknesses, Rainbow tables, Modern hashing (Argon2/bcrypt)

View Scenario 1 Launch Lab 1

Scenario 2: Insecure Cryptographic Storage

Discover how hardcoded encryption keys in source code can compromise all encrypted data.

Topics: Key management, Environment variables, Key vaults

View Scenario 2 Launch Lab 2

Scenario 3: Insufficient Randomness

Exploit predictable random number generators to compromise security tokens and sessions.

Topics: PRNG vs CSPRNG, secrets module, Token prediction

View Scenario 3 Launch Lab 3

Scenario 4: CBC Bit-Flipping Attack

Learn how encryption without integrity protection allows attackers to manipulate encrypted data.

Topics: CBC mode, MAC/HMAC, Authenticated encryption (AES-GCM)

View Scenario 4 Launch Lab 4
Learning Objectives:
  • Understand common cryptographic failures and their real-world impacts
  • Identify vulnerable code patterns in web applications
  • Apply secure cryptographic practices following OWASP guidelines
  • Recognize the importance of proper key management and algorithm selection
