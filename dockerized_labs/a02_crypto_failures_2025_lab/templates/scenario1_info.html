Scenario 1: Weak Encryption Algorithm

Vulnerability: Using MD5 for password hashing (CWE-327, CWE-916)

What is MD5?

MD5 (Message-Digest Algorithm 5) is a cryptographic hash function that produces a 128-bit hash value. While it was widely used in the past, it is now considered broken and unsuitable for security purposes.

Why is MD5 Dangerous for Passwords?

  • Speed: MD5 is extremely fast – modern GPUs can compute billions of hashes per second
  • Rainbow Tables: Precomputed lookup tables allow instant cracking of common passwords
  • Collision Attacks: Two different inputs can produce the same hash value
  • No Built-in Salt: Identical passwords always produce identical hashes, enabling bulk cracking

Real-World Impact: LinkedIn Data Breach (2012)

In 2012, LinkedIn suffered a massive breach where 6.5 million password hashes were stolen. The passwords were hashed using unsalted SHA-1 (similar weaknesses to MD5). Within days, security researchers had cracked millions of passwords using rainbow tables and brute force attacks.

Vulnerable vs Secure Code

❌ Vulnerable – MD5 Password Hashing

import hashlib

# VULNERABLE: Using MD5 for passwords
password = "mypassword123"
password_hash = hashlib.md5(password.encode()).hexdigest()
# Result: 6f8d0fca4ab0fa4b4ae7f2e4c161e0f0

# Attacker can easily crack this:
# 1. Search rainbow table
# 2. Brute force (billions of attempts per second)
# 3. Found in seconds for common passwords

✅ Secure – Argon2id Password Hashing

from argon2 import PasswordHasher

# SECURE: Use Argon2id (winner of Password Hashing Competition)
ph = PasswordHasher()
password_hash = ph.hash("mypassword123")
# Result: $argon2id$v=19$m=65536,t=3,p=4$...(long secure hash)

# Benefits:
# - Slow by design (configurable work factor)
# - Built-in salt
# - Memory-hard (resistant to GPU attacks)
# - Recommended by OWASP
